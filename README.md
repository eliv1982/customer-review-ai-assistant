# ИИ-ассистент для обработки отзывов клиентов

Учебный MVP-проект: Telegram-бот для обработки отзывов клиентов.  

Приложение принимает текстовые отзывы, сохраняет их в SQLite, анализирует через OpenAI, формирует краткую сводку и черновик ответа, подбирает подходящие шаблоны из базы знаний и строит краткий аналитический отчёт.

Проект выполнен на тестовых данных. Персональные данные не используются.

---

## Что умеет проект

- принимать отзывы через Telegram-бота;
- определять тональность и тему отзыва;
- формировать краткую сводку;
- генерировать вежливый черновик ответа клиенту;
- подбирать подходящие шаблоны из базы знаний;
- импортировать тестовые отзывы из CSV;
- строить краткий аналитический отчёт;
- защищаться от дублей при повторной отправке одинакового отзыва.

---

## Для каких задач подходит

Проект помогает автоматизировать первичную обработку обратной связи клиентов и может быть полезен для:

- служб поддержки;
- отделов качества;
- маркетинга;

---

## Архитектура

Pipeline проекта:

**Telegram / CSV → SQLite → OpenAI → knowledge base → ответ пользователю / аналитический отчёт**

Основные компоненты:

- `review_service.py` — работа с SQLite;
- `review_pipeline.py` — orchestration обработки одного отзыва;
- `ai_service.py` — вызов OpenAI и валидация структурированного ответа;
- `knowledge_base_service.py` — подбор подходящих шаблонов из CSV;
- `report_service.py` — агрегаты и текстовый отчёт;
- `bot/handlers.py` — команды и пользовательский интерфейс Telegram-бота.

## Установка зависимостей

1. Установите **Python 3.11+**.
2. Создайте виртуальное окружение:

   ```bash
   python -m venv .venv
   ```

3. Активируйте его:

   - **Windows (PowerShell):** `.\.venv\Scripts\Activate.ps1`
   - **macOS / Linux:** `source .venv/bin/activate`

4. Установите пакеты:

   ```bash
   pip install -r requirements.txt
   ```

---

## Настройка `.env`

1. Скопируйте **`.env.example`** в файл **`.env`** в корне проекта.
2. Заполните переменные:

| Переменная | Назначение |
|------------|------------|
| `TELEGRAM_BOT_TOKEN` | Токен от @BotFather — **обязателен** для работы бота |
| `OPENAI_API_KEY` | Ключ OpenAI — **обязателен** для анализа отзывов в чате |
| `OPENAI_MODEL` | Модель (по умолчанию `gpt-4o-mini`) |
| `DATABASE_PATH` | Путь к SQLite (по умолчанию `data/reviews.db`) |
| `RUN_SMOKE_TESTS` | `true` / `1` / `yes` / `on` — включить smoke **перед** ботом (по умолчанию выключено) |
| `LOG_LEVEL` | Уровень логов: `INFO`, `DEBUG`, … |

Файл **`.env`** не коммитьте в публичный репозиторий.

## Запуск бота

Из корня проекта (с активированным venv):

```bash
python main.py
```

При обычном запуске:
- инициализируется SQLite;
- запускается Telegram-бот;
- smoke tests не выполняются по умолчанию.

---

## Команды Telegram-бота

| Команда       | Назначение                  |
| ------------- | --------------------------- |
| `/start`      | краткое описание бота       |
| `/help`       | справка по командам         |
| `/new_review` | начало ввода нового отзыва  |
| `/report`     | краткий аналитический отчёт |

Любое обычное текстовое сообщение обрабатывается как отзыв.

## Как выглядит результат обработки

После отправки отзыва бот возвращает:

- тональность;
- тему;
- краткую сводку;
- черновик ответа клиенту;
- 1–2 подходящих шаблона из базы знаний.

---

## База знаний (`knowledge_base`)

Файл: data/knowledge_base.csv

База знаний используется как справочный слой и содержит:

- часто встречающиеся формулировки отзывов;
- готовые ответы по типам ситуации;
- рекомендации для оператора;
- примеры кратких сводок.

Важно: шаблоны из базы знаний не подменяют ответ модели, а дополняют его.

---

## Защита от дублей

Если один и тот же Telegram-пользователь повторно отправляет тот же текст в течение 24 часов:

- новая запись в `reviews` не создаётся;
- OpenAI не вызывается повторно;
- пользователю возвращается уже существующий результат анализа. 

---

## Аналитика

Команда /report формирует краткую сводку по данным из SQLite:

- общее количество отзывов;
- распределение по статусам;
- распределение по тональности;
- распределение по темам;
- средний рейтинг;
- проблемные темы;
- краткая сводка по продуктам.

Тестовые источники вида smoke_* исключаются из пользовательского отчёта.
